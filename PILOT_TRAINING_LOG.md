1. **CUDA не найден** → Изменен `device` с `0` на `cpu`
2. **Проблема со шрифтами** → Отключены графики (`plots: false`)
3. **Сетевые проверки PyPI** → Добавлены `ULTRALYTICS_SKIP_CHECKS`

## Что дальше

После успешного пилот-обучения:

1. **Проверить результаты**:
   - Посмотреть на логи обучения в `runs/train/yolo11n_*`, `yolo11l_*` и т.д.
   - Проверить финальные метрики в выводе

2. **Запустить полное обучение** (если пилот успешен):
   ```powershell
   python run_full_training.py
   ```
   - Это запустит полное обучение на 100 эпох для всех моделей
   - На CPU это займёт много часов!

3. **Валидировать модели**:
   ```powershell
   python run_validation.py
   ```

4. **Сравнить результаты**:
   ```powershell
   python run_comparison.py
   ```
   - Генерирует таблицы: `edge_models_comparison`, `cloud_models_comparison`, `cascade_comparison`

## Файлы для отслеживания

- **Логи обучения**: `runs/train/yolo*_YYYYMMDD_HHMMSS/`
- **Веса моделей**: `runs/train/yolo*_YYYYMMDD_HHMMSS/weights/`
  - `best.pt` — лучшие веса по валидации
  - `last.pt` — последние веса

## Примечания

- Пилот использует меньший batch size (8 вместо 16) для экономии памяти
- На CPU обучение намного медленнее, чем на GPU
- После пилота рекомендуется использовать полный конфиг с `experiment.yaml` (100 эпох)
